# AICraft Technical Implementation Manual

## 1. Project Overview

AICraft is a **Three.js**-based 3D voxel sandbox game, all code is **Vanilla JavaScript ES Module**, zero build dependencies (build only does file merging), Three.js loaded from CDN via `importmap`.

### Tech Stack

| Technology | Version/Notes |
|------|-----------|
| Three.js | 0.160.0 (CDN via unpkg) |
| Language | JavaScript (ES Modules) |
| Styles | Pure CSS (No Preprocessor) |
| Audio | Web Audio API (Procedural Synthesis) |
| Storage | localStorage |
| Build | Node.js Single-File Merge Script |

## 3. Core Systems Detailed

### 3.1 Game Loop (`main.js`)

```
requestAnimationFrame
    │
    ▼
gameLoop(time)
    │
    ├── input.update()          ← Handle input events
    ├── touchController.update()
    ├── Consume Actions (help/escape/toggleCamera/...)
    ├── player.update(dt)       ← Physics update
    ├── Update animals/slimes/hostiles
    ├── Handle combat/break/place
    ├── renderer.updateDirtyMeshes()
    ├── hud.update()            ← UI update
    └── renderer.render()       ← Three.js rendering
```

#### UI State Machine

```
gameplay ──ESC──→ pause
gameplay ──E────→ inventory
pause ────Resume──→ gameplay
inventory ─E/ESC→ gameplay
```

### 3.2 World System (`world.js`)

#### Data Structure

- **World class**: holds `blocks[width][height][depth]` 3D array
- **Block type**: 0-22 numeric enum (BLOCK constants)
- **Chunk mode**: 16×64×16 chunks, managed by ChunkManager for loading

#### Terrain Generation

```
Simplex 2D Noise (Multi-octave)
    │
    ├── Continental-scale Noise → Base terrain skeleton
    ├── Hill-scale Noise → Terrain undulation details
    ├── River noise threshold → River carving
    │
    ▼
Height map → Layered fill: Grass/Dirt/Stone
    │
    ├── Vein generation (3D Simplex)
    ├── Cave system (3D noise hollowing)
    ├── Tree/decoration generation
    └── Surface layer (Grass, flowers, cacti, etc.)
```

#### Save System

- **Incremental storage**: only records player-modified blocks (key = `"x,y,z"`)
- **Full snapshot**: includes player position, level index, skills, inventory
- **Storage location**: localStorage `aicraft_save_v4`
- **Auto-save**: every 30 seconds + on page close

### 3.3 Rendering System (`renderer.js`)

#### InstanceMesh Architecture

```
One InstancedMesh per non-air block type
    │
    ├── Legacy mode: full rebuild on update
    │
    └── Chunk mode (default):
        ├── One InstancedMesh per chunk per block type
        ├── Frustum Culling
        ├── LOD (<4 full detail / 4-8 2×2 merged / >8 hidden)
        └── Dirty flag incremental update
```

#### Environment System

```
Scene
├── Base Fog → switches to FogExp2 in swamp
├── AmbientLight
├── HemisphereLight
├── DirectionalLight + shadow map
├── Skybox color (dynamic day/night interpolation)
├── Particle system (block break fragments)
└── Dungeon lighting (PointLight array)
```

#### Day/Night System

```
timeOfDay: 0.0 → 0.25 → 0.5 → 0.75 → 1.0
            Night   Sunrise   Noon   Sunset   Night
```

Sky color interpolates between 4 key colors, light intensity changes in sync.

### 3.4 Touch Control System (`touch-controller.js`)

#### Architecture

```
TouchController
├── Joystick
│   ├── Touch area: bottom-left 35vw × 55vh
│   ├── Displacement normalized [-1, 1], dead zone 0.25
│   └── Maps to WASD key state
│
├── Camera Drag
│   ├── Touch area: top-right 40vw × 60vh
│   └── Increment accumulates to input.mouseDelta
│
├── Action Buttons
│   ├── Jump/Break(long-press)/Place/Dodge
│   └── Break long-press triggers continuously at 200ms interval
│
├── Func Buttons
│   ├── Interact/Skill/Switch/Weapon/Fly/Camera/Help
│   └── 3×2 grid + Help centered
│
└── Pause button (top-left)
```

#### Button Event Binding

```
touchstart → _handleActionStart(action)
    ├── break: mouseButtons[0]=true + 200ms interval
    ├── place: mouseButtons[2]=true + push 'place'
    ├── jump/dodge/skill/...: push action string
    └── escape: push 'escape'

touchend → _handleActionEnd(action)
    ├── break: clear interval + mouseButtons[0]=false
    └── place: mouseButtons[2]=false
```

### 3.5 Collision System (`player.js`)

#### Physics Parameters

| Parameter | Value |
|------|-----|
| Gravity | -25 m/s² |
| Jump Speed | 8 m/s |
| Max Jump Height | ~1.25 blocks |
| Movement Speed | 4.3 blocks/s |
| Hitbox | 0.6W × 1.8H |
| Eye Height | 1.6 |

#### Collision Detection

```
Per-axis detection (X → Y → Z independent fallback)
    │
    ├── Compute moved AABB
    ├── Check voxels covered by AABB
    ├── If overlap → fall back along that axis
    └── Process three axes sequentially
```

### 3.6 Raycasting (`raycast.js`)

3D DDA (Digital Differential Analyzer) algorithm:

```
1. Cast ray from camera/eye position
2. Step along ray to detect voxels
3. On non-air block → return position + normal
4. Max distance 8 blocks (god mode 60 blocks)
```

### 3.7 Input System (`input.js`)

#### Architecture

```
Input
├── Keyboard state keys map
├── Mouse buttons mouseButtons[]
├── Mouse delta mouseDeltaX/Y
├── Action queue _actions[]
│
├── consumeAction(name)      ← Consume and remove action
├── isKeyDown(code)          ← Key held state
├── requestLock()            ← Pointer Lock API
└── setTouchMode(bool)       ← Touch mode flag
```

#### Actions Mapping

All interactions (keyboard + touch) are uniformly converted to action strings, placed in the `_actions[]` queue, and consumed by the game loop via `consumeAction()`.

### 3.8 Audio System (`audio.js`)

#### Procedural Synthesis

```
AudioManager
├── BGM: Multi-layer 32s loop
│   ├── Melody layer (square/sawtooth oscillator)
│   ├── Chord layer
│   ├── Bass layer
│   ├── Percussion layer
│   └── Ambient layer
│
├── SFX: 15 sound effects
│   ├── Uses OscillatorNode + GainNode
│   ├── Frequency envelope + volume envelope
│   └── Types: break/place/attack/hurt/jump/pickup etc.
│
├── ConvolverNode reverb
└── StereoPannerNode spatial positioning
```

### 3.9 Hostile Mob System (`hostiles.js`)

#### Mob Types and AI

| Mob | AI Behavior | Attack Method |
|------|---------|----------|
| Zombie | Tracks player, burns in sunlight | Melee |
| Skeleton | Keeps distance, shoots | Ranged arrows |
| Spider | Fast tracking | Melee |
| Cave Spider | Fast tracking | Melee + Poison |
| Creeper | Approaches then explodes | Area explosion |
| Enderman | Teleports, enrages on hit | Melee |
| Wolf | Pack tracking | Melee |
| Slime | Hopping track | Contact damage |

#### Spawn System

```
_spawnHostiles()
    ├── Attempt spawn every 8 seconds
    ├── Max 16
    ├── Probability halved during day
    ├── 15-30 block range from player
    └── Biome weight adjustment
```

## 4. Build Process

```
build.js
    │
    ├── Read index.html
    ├── Read all <script type="module" src="*.js">
    ├── Read all CSS <link>
    ├── Inline JS code into HTML
    ├── Inline CSS into HTML
    ├── Copy image files
    └── Output dist/AICraft.html (single file)
```

The build output is a **standalone single-file HTML** that can be opened in any browser without a server.

## 5. AI Development Workflow

### OpenSpec Workflow

```
1. Propose → proposal.md
   ├── Describe requirements and motivation
   └── List affected capabilities

2. Spec → spec.md
   ├── Precise capability specification
   └── Input/Output/Constraints

3. Design → design.md
   ├── Architecture decisions
   ├── Trade-off analysis
   └── ASCII diagrams

4. Tasks → tasks.md
   ├── Executable task list
   └── Each task no more than 2 hours

5. Apply → AI auto-implementation
   ├── Read design documents
   ├── Complete tasks one by one
   └── Build verification

6. Commit → Commit to Git
```

### Development Principles

- **Incremental development**: change one capability at a time, iterate quickly
- **Documentation-driven**: write design first, then code
- **Auto-validation**: auto-build after each change
- **Chinese-first**: all documentation and comments in Chinese

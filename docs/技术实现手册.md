# AICraft 技术实现手册

## 一、项目概述

AICraft 是一个基于 **Three.js** 的 3D 体素沙盒游戏，所有代码均为 **Vanilla JavaScript ES Module**，零构建依赖（构建仅做文件合并），通过 `importmap` 从 CDN 加载 Three.js。

### 技术栈

| 技术 | 版本/说明 |
|------|-----------|
| Three.js | 0.160.0 (CDN via unpkg) |
| 语言 | JavaScript (ES Modules) |
| 样式 | 纯 CSS（无预处理器） |
| 音效 | Web Audio API（程序化合成） |
| 存储 | localStorage |
| 构建 | Node.js 单文件合并脚本 |

## 三、核心系统详解

### 3.1 游戏主循环 (`main.js`)

```
requestAnimationFrame
    │
    ▼
gameLoop(time)
    │
    ├── input.update()          ← 处理输入事件
    ├── touchController.update()
    ├── 消费 Actions（help/escape/toggleCamera/...）
    ├── player.update(dt)       ← 物理更新
    ├── 更新动物/史莱姆/敌对生物
    ├── 处理战斗/破坏/放置
    ├── renderer.updateDirtyMeshes()
    ├── hud.update()            ← UI 更新
    └── renderer.render()       ← Three.js 渲染
```

#### UI 状态机

```
gameplay ──ESC──→ pause
gameplay ──E────→ inventory
pause ────继续───→ gameplay
inventory ─E/ESC→ gameplay
```

### 3.2 世界系统 (`world.js`)

#### 数据结构

- **World 类**：持有 `blocks[width][height][depth]` 三维数组
- **方块类型**：0-22 数字枚举（BLOCK 常量）
- **分块模式**：16×64×16 区块，ChunkManager 管理加载

#### 地形生成

```
Simplex 2D 噪声（多层级）
    │
    ├── 大陆尺度噪声 → 基础地形骨架
    ├── 丘陵尺度噪声 → 地形起伏细节
    ├── 河流噪声阈值 → 河流切割
    │
    ▼
高度图 → 分层填充：草/泥土/石头
    │
    ├── 矿脉生成（3D Simplex）
    ├── 洞穴系统（3D 噪声挖空）
    ├── 树木/装饰生成
    └── 地表层（草地、花朵、仙人掌等）
```

#### 存档系统

- **增量存储**：仅记录玩家修改过的方块（key = `"x,y,z"`）
- **完整快照**：包含玩家位置、关卡索引、技能、背包
- **存储位置**：localStorage `minicraft_save_v4`
- **自动保存**：每 30 秒 + 关闭页面时

### 3.3 渲染系统 (`renderer.js`)

#### InstanceMesh 架构

```
每种非空气方块类型一个 InstancedMesh
    │
    ├── 传统模式：全量构建，更新时重建
    │
    └── 分块模式（默认）：
        ├── 每区块每方块类型一个 InstancedMesh
        ├── 视锥体裁剪（Frustum Culling）
        ├── LOD（<4 全细节 / 4-8 2×2 合并 / >8 隐藏）
        └── 脏标记增量更新
```

#### 环境系统

```
场景
├── 基础雾 (Fog) → 沼泽时切换 FogExp2
├── 环境光 (AmbientLight)
├── 半球光 (HemisphereLight)
├── 方向光 (DirectionalLight) + 阴影贴图
├── 天空盒颜色（昼夜动态插值）
├── 粒子系统（破坏方块碎片）
└── 地下城灯光（PointLight 阵列）
```

#### 昼夜系统

```
timeOfDay: 0.0 → 0.25 → 0.5 → 0.75 → 1.0
            黑夜   日出   正午   日落   黑夜
```

天空颜色在 4 个关键色之间插值，光照强度同步变化。

### 3.4 触摸控制系统 (`touch-controller.js`)

#### 架构

```
TouchController
├── 摇杆（Joystick）
│   ├── 触摸区域：左下 35vw × 55vh
│   ├── 位移归一化 [-1, 1]，死区 0.25
│   └── 映射 WASD 按键状态
│
├── 镜头拖拽（Camera Drag）
│   ├── 触摸区域：右上 40vw × 60vh
│   └── 增量累加到 input.mouseDelta
│
├── 动作按钮（Action Buttons）
│   ├── 跳/挖(长按)/放/闪
│   └── 挖长按 @200ms 间隔持续触发
│
├── 功能按钮（Func Buttons）
│   ├── 交互/技能/切换/武器/飞行/视角/帮助
│   └── 3×2 网格 + 帮助居中
│
└── 暂停按钮（左上角）
```

#### 按钮事件绑定

```
touchstart → _handleActionStart(action)
    ├── break: mouseButtons[0]=true + 200ms interval
    ├── place: mouseButtons[2]=true + push 'place'
    ├── jump/dodge/技能/...: push action string
    └── escape: push 'escape'

touchend → _handleActionEnd(action)
    ├── break: 清除 interval + mouseButtons[0]=false
    └── place: mouseButtons[2]=false
```

### 3.5 碰撞系统 (`player.js`)

#### 物理参数

| 参数 | 值 |
|------|-----|
| 重力 | -25 m/s² |
| 跳跃速度 | 8 m/s |
| 最高跳跃 | ~1.25 格 |
| 移动速度 | 4.3 格/秒 |
| 碰撞箱 | 0.6W × 1.8H |
| 眼睛高度 | 1.6 |

#### 碰撞检测

```
分轴检测（X → Y → Z 独立回退）
    │
    ├── 计算移动后的 AABB
    ├── 检查 AABB 覆盖的体素
    ├── 若有重叠 → 沿该轴回退
    └── 三个轴依次处理
```

### 3.6 射线检测 (`raycast.js`)

3D DDA（数字微分分析）算法：

```
1. 从摄像机/眼睛位置发射射线
2. 沿射线步进检测体素
3. 遇非空气方块 → 返回位置+法线
4. 最大距离 8 格（上帝模式 60 格）
```

### 3.7 输入系统 (`input.js`)

#### 架构

```
Input
├── 键盘状态 keys 映射
├── 鼠标按钮 mouseButtons[]
├── 鼠标增量 mouseDeltaX/Y
├── Action 队列 _actions[]
│
├── consumeAction(name)      ← 消费并移除动作
├── isKeyDown(code)          ← 按键持续状态
├── requestLock()            ← Pointer Lock API
└── setTouchMode(bool)       ← 触摸模式标记
```

#### Actions 映射

所有交互（键盘 + 触摸）统一转换为 action 字符串，放入 `_actions[]` 队列，主循环通过 `consumeAction()` 消费。

### 3.8 音频系统 (`audio.js`)

#### 程序化合成

```
AudioManager
├── BGM: 多层 32s 循环
│   ├── 旋律层（方波/锯齿波振荡器）
│   ├── 和弦层
│   ├── 低音层
│   ├── 打击层
│   └── 环境层
│
├── SFX: 15 种音效
│   ├── 使用 OscillatorNode + GainNode
│   ├── 频率包络 + 音量包络
│   └── 类型：破坏/放置/攻击/受伤/跳跃/拾取等
│
├── ConvolverNode 混响
└── StereoPannerNode 空间定位
```

#### 生物类型与 AI

| 蜘蛛 | 快速追踪 | 近战 |

| 史莱姆 | 跳跃追踪 | 接触伤害 |

#### 生成系统

```

    ├── 每 8 秒尝试生成
    ├── 上限 16 只
    ├── 白天概率减半
    ├── 距离玩家 15-30 格范围
    └── 生物群系权重调整
```

## 四、构建流程

```
build.js
    │
    ├── 读取 index.html
    ├── 读取所有 <script type="module" src="*.js">
    ├── 读取所有 CSS <link>
    ├── 内联 JS 代码到 HTML
    ├── 内联 CSS 到 HTML
    ├── 复制图片文件
    └── 输出 dist/Minicraft.html（单文件）
```

构建产物为**独立单文件 HTML**，无需服务器即可在浏览器中打开。

## 五、AI 开发流程

### OpenSpec 工作流

```
1. Propose → proposal.md
   ├── 描述需求和动机
   └── 列出影响的能力

2. Spec → spec.md
   ├── 精确的能力规格
   └── 输入/输出/约束

3. Design → design.md
   ├── 架构决策
   ├── 权衡分析
   └── ASCII 图表

4. Tasks → tasks.md
   ├── 可执行的任务列表
   └── 每项不超过 2 小时

5. Apply → AI 自动实现
   ├── 读取设计文档
   ├── 逐一完成任务
   └── 构建验证

6. Commit → 提交到 Git
```

### 开发原则

- **增量开发**：每次只改一个能力，小步快跑
- **文档驱动**：先写设计，再写代码
- **自动验证**：每次修改后自动构建
- **中文优先**：所有文档和注释使用中文

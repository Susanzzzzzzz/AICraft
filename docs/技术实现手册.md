# AICraft 技术实现手册

## 1. 项目概述

AICraft 是一款基于 **Three.js** 的 3D 体素沙盒游戏，全部代码为 **Vanilla JavaScript ES Module**，零构建依赖（构建仅做文件合并），Three.js 通过 `importmap` 从 CDN 加载。

### 技术栈

| 技术 | 版本/说明 |
|------|-----------|
| Three.js | 0.160.0（CDN via unpkg） |
| 语言 | JavaScript（ES Modules） |
| 样式 | 纯 CSS（无预处理器） |
| 音频 | Web Audio API（程序化合成） |
| 存储 | localStorage |
| 构建 | Node.js 单文件合并脚本 |

## 3. 核心系统详解

### 3.1 游戏主循环（`main.js`）

```
requestAnimationFrame
    │
    ▼
gameLoop(time)
    │
    ├── input.update()          ← 处理输入事件
    ├── touchController.update()
    ├── 消费动作（帮助/退出/切视角/...）
    ├── player.update(dt)       ← 物理更新
    ├── 更新动物/史莱姆/敌对生物
    ├── 处理战斗/破坏/放置
    ├── renderer.updateDirtyMeshes()
    ├── hud.update()            ← UI 更新
    └── renderer.render()       ← Three.js 渲染
```

#### UI 状态机

```
gameplay ──ESC──→ pause（暂停）
gameplay ──E────→ inventory（背包）
pause ────继续──→ gameplay
inventory ─E/ESC→ gameplay
```

### 3.2 世界系统（`world.js`）

#### 数据结构

- **World 类**：持有 `blocks[width][height][depth]` 三维数组
- **方块类型**：0-22 数字枚举（BLOCK 常量）
- **区块模式**：16×64×16 区块，由 ChunkManager 管理加载

#### 地形生成

```
Simplex 2D 噪声（多倍频）
    │
    ├── 大陆尺度噪声 → 基础地形骨架
    ├── 丘陵尺度噪声 → 地形起伏细节
    ├── 河流噪声阈值 → 河流雕刻
    │
    ▼
高度图 → 分层填充：草/土/石头
    │
    ├── 矿脉生成（3D Simplex）
    ├── 洞穴系统（3D 噪声掏空）
    ├── 树木/装饰物生成
    └── 地表层（草、花、仙人掌等）
```

#### 存档系统

- **增量存储**：仅记录玩家修改过的方块（key = `"x,y,z"`）
- **完整快照**：包括玩家位置、关卡索引、技能、背包
- **存储位置**：localStorage `aicraft_save_v4`
- **自动存档**：每 30 秒 + 页面关闭时

### 3.3 渲染系统（`renderer.js`）

#### InstanceMesh 架构

```
每种非空气方块一个 InstancedMesh
    │
    ├── 传统模式：更新时全量重建
    │
    └── 区块模式（默认）：
        ├── 每个区块每种方块一个 InstancedMesh
        ├── 视锥体裁剪
        ├── LOD（<4 全细节 / 4-8 2×2 合并 / >8 隐藏）
        └── 脏标记增量更新
```

#### 环境系统

```
场景
├── 基础雾 → 沼泽中切换为 FogExp2
├── AmbientLight
├── HemisphereLight
├── DirectionalLight + 阴影贴图
├── 天空盒颜色（动态昼夜插值）
├── 粒子系统（方块破坏碎片）
└── 地牢照明（PointLight 数组）
```

#### 昼夜系统

```
timeOfDay: 0.0 → 0.25 → 0.5 → 0.75 → 1.0
            夜晚    日出   正午   日落    夜晚
```

天空颜色在 4 个关键色间插值，光照强度同步变化。

### 3.4 触屏控制系统（`touch-controller.js`）

#### 架构

```
TouchController
├── 摇杆
│   ├── 触摸区域：左下 35vw × 55vh
│   ├── 位移归一化 [-1, 1]，死区 0.25
│   └── 映射到 WASD 按键状态
│
├── 镜头拖拽
│   ├── 触摸区域：右上 40vw × 60vh
│   └── 增量累积到 input.mouseDelta
│
├── 操作按钮
│   ├── 跳跃/挖掘（长按）/放置/闪避
│   └── 挖掘长按以 200ms 间隔持续触发
│
├── 功能按钮
│   ├── 交互/技能/切换/武器/飞行/视角/帮助
│   └── 3×2 网格 + 帮助居中
│
└── 暂停按钮（左上角）
```

#### 按钮事件绑定

```
touchstart → _handleActionStart(action)
    ├── break: mouseButtons[0]=true + 200ms 定时器
    ├── place: mouseButtons[2]=true + 推送 'place'
    ├── jump/dodge/skill/...: 推送动作字符串
    └── escape: 推送 'escape'

touchend → _handleActionEnd(action)
    ├── break: 清除定时器 + mouseButtons[0]=false
    └── place: mouseButtons[2]=false
```

### 3.5 碰撞系统（`player.js`）

#### 物理参数

| 参数 | 值 |
|------|-----|
| 重力 | -25 m/s² |
| 跳跃速度 | 8 m/s |
| 最大跳跃高度 | ~1.25 格 |
| 移动速度 | 4.3 格/s |
| 碰撞箱 | 0.6 宽 × 1.8 高 |
| 眼高 | 1.6 |

#### 碰撞检测

```
逐轴检测（X → Y → Z 独立回退）
    │
    ├── 计算移动后的 AABB
    ├── 检测 AABB 覆盖的体素
    ├── 若重叠 → 沿该轴回退
    └── 三个轴依次处理
```

### 3.6 射线检测（`raycast.js`）

3D DDA（数字微分分析）算法：

```
1. 从相机/眼睛位置发射射线
2. 沿射线步进检测体素
3. 遇到非空气方块 → 返回位置 + 法线
4. 最大距离 8 格（上帝模式 60 格）
```

### 3.7 输入系统（`input.js`）

#### 架构

```
Input
├── 键盘状态 keys 映射
├── 鼠标按键 mouseButtons[]
├── 鼠标增量 mouseDeltaX/Y
├── 动作队列 _actions[]
│
├── consumeAction(name)      ← 消费并移除动作
├── isKeyDown(code)          ← 按键保持状态
├── requestLock()            ← Pointer Lock API
└── setTouchMode(bool)       ← 触屏模式标志
```

#### 动作映射

所有交互（键盘 + 触屏）统一转为动作字符串，放入 `_actions[]` 队列，游戏主循环通过 `consumeAction()` 消费。

### 3.8 音频系统（`audio.js`）

#### 程序化合成

```
AudioManager
├── BGM：多层 32 秒循环
│   ├── 旋律层（方波/锯齿波振荡器）
│   ├── 和弦层
│   ├── 低音层
│   ├── 打击乐层
│   └── 环境层
│
├── SFX：15 种音效
│   ├── 使用 OscillatorNode + GainNode
│   ├── 频率包络 + 音量包络
│   └── 类型：破坏/放置/攻击/受伤/跳跃/拾取等
│
├── ConvolverNode 混响
└── StereoPannerNode 空间定位
```

## 4. 构建流程

```
build.js
    │
    ├── 读取 index.html
    ├── 读取所有 <script type="module" src="*.js">
    ├── 读取所有 CSS <link>
    ├── 将 JS 代码内联到 HTML
    ├── 将 CSS 内联到 HTML
    ├── 复制图片文件
    └── 输出 dist/AICraft.html（单文件）
```

构建产物为**独立的单文件 HTML**，无需服务器即可在任何浏览器中打开。

## 5. AI 开发工作流

### OpenSpec 工作流

```
1. 提案 → proposal.md
   ├── 描述需求和动机
   └── 列出受影响的模块

2. 规格 → spec.md
   ├── 精确的功能规格说明
   └── 输入/输出/约束条件

3. 设计 → design.md
   ├── 架构决策
   ├── 权衡分析
   └── ASCII 图表

4. 任务 → tasks.md
   ├── 可执行的任务列表
   └── 每个任务不超过 2 小时

5. 实现 → AI 自动编码
   ├── 阅读设计文档
   ├── 逐个完成任务
   └── 构建验证

6. 提交 → 提交到 Git
```

### 开发原则

- **增量开发**：一次修改一个功能，快速迭代
- **文档驱动**：先写设计，再写代码
- **自动验证**：每次修改后自动构建
- **中文优先**：所有文档和注释使用中文

# Mount System — 规格

## 能力描述

管理坐骑 (马) 的模型、AI、骑乘交互和物理变化。

## 输入

- 交互输入: E 键 (骑乘/下马), Shift 键 (下马)
- 移动输入: WASD (骑乘时控制坐骑)
- 帧时间: dt

## 输出

- 骑乘状态: isRiding, mountRef
- 坐骑位置/速度 (骑乘时覆盖玩家物理)

## 功能要求

### M-1: 马模型
- Voxel 风格 BoxGeometry 组合:
  | 部位 | 尺寸 | 位置偏移 |
  |------|------|---------|
  | body | 1.4×0.8×0.6 | y=0.8 |
  | head | 0.5×0.5×0.4 | x=0.7, y=1.2, rotX=-30° |
  | neck | 0.3×0.6×0.3 | x=0.5, y=1.1 |
  | leg_FL | 0.2×0.6×0.2 | x=0.5, z=0.2 |
  | leg_FR | 0.2×0.6×0.2 | x=0.5, z=-0.2 |
  | leg_BL | 0.2×0.6×0.2 | x=-0.5, z=0.2 |
  | leg_BR | 0.2×0.6×0.2 | x=-0.5, z=-0.2 |
  | tail | 0.15×0.5×0.15 | x=-0.8, y=1.0, rotX=45° |
- 颜色: 棕色 #8B4513 为主, 深棕 #654321 细节

### M-2: 马行走动画
- 对角步态:
  ```
  swing = Math.sin(time * 6) * 0.5
  leg_FL_pivot.rotation.x = swing
  leg_BR_pivot.rotation.x = swing
  leg_FR_pivot.rotation.x = -swing
  leg_BL_pivot.rotation.x = -swing
  ```
- idle 动画: 尾巴轻微摆动, 头轻微上下

### M-3: 马 AI
- 三状态:
  - **idle**: 原地不动, 偶尔随机转头 (概率 0.5%/帧)
  - **wander**: 向随机方向走 3-5 秒, 然后回到 idle
  - **flee**: 受击后向反方向快速跑 3 秒
- 状态转换:
  ```
  idle → wander: 随机触发 (1%/帧)
  wander → idle: 走完预定时间
  idle/wander → flee: 受击
  flee → idle: 跑完 3 秒
  被骑 → 所有 AI 暂停
  ```
- 碰撞: 马的 AI 移动也受世界碰撞约束

### M-4: 马的生成
- 世界生成后, 在平坦区域 (高度差 ≤ 1 的连续 3×3 区域) 随机放置 2-3 匹马
- 马的初始状态: idle, 随机朝向

### M-5: 骑乘交互
- 上马条件:
  - 玩家与马距离 < 2 格
  - 玩家不在骑乘状态
  - 按 E 键或右键点击马
- 上马效果:
  - player.isRiding = true
  - player.mountRef = horse
  - 玩家位置吸附到马背上
  - 马的 AI 暂停
- 下马条件:
  - 按 Shift 键
  - 按 E 键 (切换)
  - 马死亡 (如有)
- 下马效果:
  - player.isRiding = false
  - 玩家位置偏移到马侧面
  - 马的 AI 恢复

### M-6: 骑乘物理
- 骑乘时物理参数变化:
  | 参数 | 步行 | 骑马 |
  |------|------|------|
  | 碰撞箱宽度 | 0.6 | 1.0 |
  | 碰撞箱高度 | 1.8 | 2.2 |
  | 移动速度 | 4.3 | 9.0 |
  | 跳跃高度 | 1.25 | 2.0+ |
  | 眼睛高度 | 1.6 | 2.0 |
- WASD → 控制马的方向和移动
- Space → 马跳跃 (使用马的跳跃力)
- 碰撞检测使用马的碰撞箱
- 玩家位置 = 马的位置 + 骑乘偏移 (马背上)

### M-7: 骑乘摄像机
- 第一人称: camera 在马背上, 高度 = 马背 + 1.6
- 第三人称: camera 在马后方, 偏移更大 (马比人宽)
- raycast 起点从马背上发射
- 射线排除马模型 (layers)

## 约束

- 同一时间只能骑一匹马
- 一匹马同时只能被一个人骑 (单人游戏自动满足)
- 马不能进 1 格宽通道 (宽度 1.0), 这是有意设计 → 增加下马探索的游戏性
- 马受碰撞约束, 不会穿墙
- 水中马会浮起来 (简化: 水面高度 < 马背时马可进入)

## 依赖

- Three.js (BoxGeometry, Group)
- player.js (骑乘状态)
- world.js (碰撞检测)
- camera.js (视角适配)

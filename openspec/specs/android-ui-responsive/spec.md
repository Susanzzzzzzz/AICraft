# Android UI 响应式布局

## 能力描述

在 Android 横屏设备（以及小屏幕移动端浏览器）上，所有非游戏进行中界面（开始界面、暂停界面、背包界面、宝箱界面、关卡选择、通关通知）能够自动适配 360-430px 高度的窄视口，内容不再超出屏幕或被截断。

## 触发条件

- 自动生效，基于 CSS `@media` 查询响应视口尺寸
- 横屏高度触发：`@media (max-height: 450px)` — Android 横屏典型高度
- 宽度触发：`@media (max-width: 768px)` 和 `@media (max-width: 480px)` — 已有规则补充完善

## 问题分析

Android 手机横屏 CSS 像素尺寸约为 **640-800 × 360-430px**，当前各界面以桌面 800×600+ 视口设计，高度严重超标：

| 界面 | 内容高度 | Android 可用高度 | 溢出 |
|------|---------|----------------|------|
| 开始界面 | ~580px | ~400px | ~180px |
| 暂停界面 | ~500px+ | ~400px | ~100px+ |
| 背包界面 | ~500px+ | ~400px | ~100px+ |
| 宝箱界面 | ~230px | ~400px | 基本适配 |
| 关卡选择 | ~300px | ~400px | 可能刚好 |
| 通关通知 | ~200px | ~400px | 没问题 |

## 功能规格

### 通用兜底规则

所有 overlay 容器统一添加溢出滚动支持：

```css
#overlay, #inventory-overlay, #chest-overlay,
#level-complete-notification {
  overflow-y: auto;
  -webkit-overflow-scrolling: touch; /* iOS 顺滑滚动 */
}
```

Overlay 内容区域在高度不足时应对齐到顶部（`align-items: flex-start`）而非垂直居中，避免底部内容不可见。

### 开始界面 (max-height: 450px)

```
┌─────────────────────────────┐
│  AICRAFT                    │  ← h1 22px
│  一个体素沙盒游戏            │  ← subtitle 12px
│  ┌───────────────────────┐  │
│  │ WASD 移动     ↑↓←→   │  │  ← 控制说明两列，行高 16px
│  │ 跳跃        Space    │  │
│  │ ...                   │  │
│  └───────────────────────┘  │
│       [开始游戏]            │  ← 按钮缩小
└─────────────────────────────┘
```

- 隐藏左侧角色照片（已有 <768px 规则）
- h1 字体缩小至 22-24px（比现有 <480px 的 28px 更小）
- 控制说明区域：`font-size: 10px; line-height: 16px; padding: 4px 8px`
- 控制行间距压缩，两列之间的 `gap` 缩小
- `#start-btn`：`padding: 6px 24px; font-size: 14px`
- 整体 padding 从 20px 缩减到 10-12px


- 隐藏右侧角色预览（已有 <480px 规则）
- h1 字体缩小至 18-20px
- 角色卡片：`padding: 4px 6px; font-size: 11px`，头像缩小至 28px
- `#pause-actions button`：`padding: 6px 12px; font-size: 12px`
- `#pause-volume`：更紧凑，gap 缩小
- 灵敏度滑块宽度缩至 60px
- `.pause-content` gap 从 24px 缩至 8px

- `.inventory-container` padding 从 20px 缩至 8-10px

- 存储网格：`repeat(6, 32px)` 格子缩小
- `inv-slot`：32px × 32px（已存在于 <480px）
- 合成面板默认折叠（已有逻辑）
- 装备栏垂直→水平（已存在于 <480px）

### 宝箱界面 (max-height: 450px)

- 已有 <768px 规则覆盖率足够
- 仅在 <480px 时补充更小的 padding

### 关卡选择 (max-height: 450px)

- h1 缩小至 22px
- 关卡卡片 padding 缩小至 8px，font-size 缩小
- 网格 gap 缩小

### 关卡通关通知 (max-height: 450px)

- 已有 `max-width: 90vw; min-width: unset` 规则
- padding 缩小，h2 缩小

### 现有 <768px 规则补全

当前 `<768px` 和 `<480px` 已有大量适配规则，但存在以下缺口：

1. **宝箱界面 `<480px` 无专属规则** — 虽可继承 `<768px` 规则，但在 480px 以下手机上仍需更紧凑的 padding
2. **缺少 `@media (max-height: 450px)` 横屏高度查询** — 所有高度压缩逻辑应集中在此
3. **#overlay 在 `<768px` 下的滚动** — 已验证在 [style.css:1360-1364](src/style.css#L1360-L1364) 已添加

### 优先级

1. **P0** — 添加 `@media (max-height: 450px)` 块，覆盖开始/暂停/背包三大界面
2. **P1** — 宝箱界面补充 `<480px` 规则
3. **P2** — 在真实 Android 设备上验证所有界面

## 验收标准

1. Android 横屏（732×412 典型尺寸）下，开始界面完整可见，无需滚动即可看到标题 + 控制说明 + 开始按钮
2. Android 横屏下暂停界面完整可见，角色卡 + 操作按钮 + 音量控制全部不溢出
3. Android 横屏下背包界面能完整显示网格 + 合成 + 装备栏
4. 所有 overlay 在内容超长时有 `overflow-y: auto` 兜底
5. 桌面端不受影响（高度查询仅在 max-height 触发时生效）
